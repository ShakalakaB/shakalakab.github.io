我就这样成为了一名程序员

Posted on 2019-11-02 Edited on 2020-05-01

2019年4月底从产品岗位辞职，5月、6月两个月窝着写代码，完成第一个静态网站作品后便开始找前端岗位却误打误撞进入后端，成为PHP工程师，以此开启了我的编程职业生涯。
这个经历连我自己想起来都觉得“额，有点莫名啊”，我自己速问速答记录一下这段旅程的一些关键点。

Q: 为什么从产品岗位离职去做编程？
A: 从小崇拜黑客，在Free Code Camp试水后确定喜欢编程，适合编程，一些性格特质也非常适合，外加当时工作中一些不爽的事情加快了辞职进程。

Q: 为什么学的是前端，找的是后端？
A: 这个属于不在剧本编排中的情节。我正式开始找编程工作后，确实是一心冲前端去的。但所有面试中我都如实说了，未来计划是从前端走向后端，拥有比较全面的技术能力。
面试现公司时刚好聊得投缘，刚好他们有空缺的后端岗位，而我的前端也只是开始，所以就直接问了我 “反正都是刚开始，你敢不敢试下后端”， 我就直接在一个前端offer和后端offer中选了后端。
技术经理当时和我说这对我们彼此都像是赌博，我想能让他愿意赌一把的原因大概是： 1.我通过全英文资料快速自学的过程， 2.我的学历， 3.坦诚冷静沟通

Q: 如何自学编程？
A: 我的编程自学并不算通畅，印象我大学阶段应该有至少3次尝试自学编程，但都因不得要领而草草放弃。如今回想几次失败的根本原因是：
当年是通过阅读学习编程，就好像我人生中所有绝大多数学习一样，我先找来书，翻出视频，甚至我找到编程学习网站，然后我开始“阅读”，开始“看”。
但编程是一项应用型技能，应用型技能的最佳学习方式是应用，不是阅读，因此编程最好的学习方式是赶紧开始打代码，赶紧想一个你要通过编程建造的“东西”，网站、爬虫、哪怕一个简单的网页，总之，build somethig。
除此外，几个加速进入正确轨道的窍门是：

  1. 读英文资料，用英文搜索，英文资料的质量会帮你按下加速学习键，用英文去获取世间大多数信息；
  2. 用编程去赚钱，你所面对的编程问题复杂度很大程度决定了你的技术进化速度，你可以通过建立网站开启编程第一步，但如果不进入实战环境，你将很难有还需要编程解决的问题，也很难拥有解决相应问题所需的资源，你无法靠自我模拟获得真实的编程问题和资源，自此你的编程也将停止。
  3. 多加入程序员社区，结交程序员，如果关系好或者时机适当，说出你的目标，你的进度，你需要的帮助。

Q: 作为女生，学编程或者做程序员有什么不同吗？
A：程序员会因为你是女生而倾向于帮助你（无法否认必须承认），适当运用这种帮助能让你更快成长。
当然也会有人告诉你，程序员加班多，我作为HR更想要更耐操的男码农，或者女程序员升职可能面临偏见。对此，回一句F***K最好。
同时，我朝夕相处的同事几乎都是男的，这个会带来一些相比以往不太一样的社交环境，不要太在意，代码写好比较重要。
一句话，不要太把自己的性别当回事，除非你真的觉得这是个事。

Q: 这一路最下来有什么感触?
A: 我原来看过一句话貌似是”Don’t let better be the enemy of good.”,大概就是说”别让更好成为好的敌人”,做事情的时候,不要因为想做的更好反而让自己不能开始做好。当开始准备转行时，很容易有一种心理：我要准备充分，我要把Free Code Camp上的题都做了再开始准备自己作品，我要把自己的作品做的更好再开始去找工作，我要有更多的作品再去开始…
不管是因为对未知的恐惧或是对更高标准的追求，总是要万无一失才敢走出下一步的人生是很难在人生弯道里超车的，也难以获得意料之外的人生惊喜。有时候越是重大的选择，重要的那一步反而不会是深思熟虑，精心准备好才跨出的，有时候就是无知无畏，就是“老子玩得起”的混混态度才可以做出的。
我也经历过总想等待自己做得更好再开始下一步的心理，但庆幸当时心血来潮去和认识的程序员随便聊了聊，当他们告诉我下一步的时候也侧面给了我没有十足准备也要前行的勇气。只学了Free Code Camp上前几章也可以开始准备自己作品，作品很少、作品还有一些小bug、经历很不相关，也要开始去面试，去碰撞，去发现自己要补足的。
当你通向未知时，如果没有溺水挣扎的瞬间证明你方向有误。
